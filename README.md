## Introduction
Software defect introduction is a well-studied problem in empirical software engineering, yet the relationship between developer commit patterns and code quality remains largely unexamined. While engineering culture is rife with cautionary tales about late-night coding sessions, there is surprisingly little large-scale empirical evidence linking time of day to measurable differences in defect rates. This study attempt to address that gap by mining commit histories from open source GitHub repositories to investigate whether commits made during late-night hours (00:00–04:00 local time) exhibit a statistically significantly higher defect introduction rate than those made during standard daytime hours (09:00–17:00). Specifically, this research asks: (1) Is time of day a statistically significant predictor of defect-inducing commits, and does this effect survive after controlling for confounds such as developer experience, repository size, and programming language? (2) Do late-night commits exhibit measurably different linguistic characteristics in their commit messages, such as shorter length or lower readability scores?
